## 传输安全：你的下载过程被监控了吗？

这是很多人最关心的问题，尤其是经常下 BT 资源的用户。

**好消息是：你的 IP 不会暴露在 BT 网络中。**

PikPak 的云下载机制决定了，下载行为发生在 PikPak 的远程服务器上，不是在你的设备上。从 BT 网络的角度来看，参与下载的是 PikPak 的服务器 IP，而不是你家的网络地址。你在 BT Tracker 上是完全隐形的。

对于担心 ISP 监控的用户，PikPak 也给出了说法：**所有服务器通信均使用加密协议**，ISP 无法直接监控你通过 PikPak 保存了什么内容。

有用户在论坛上也讨论过这个问题，结论是 PikPak 走的是 HTTPS/PCDN，并非传统 P2P 架构，偶发的少量上传流量是正常的 CDN 传输，不存在 P2P 泄露真实 IP 的情况。

## 数据存储安全：你的文件，PikPak 自己能看到吗？

这里有一个很重要的区别，必须说清楚。

PikPak 使用的是 **SSL/TLS 加密**传输——也就是说，数据在你和服务器之间传输的过程是加密的，没有人能在中途截获。

但，**PikPak 并不提供端对端加密（E2E Encryption）**。

这意味着什么？意味着存储在 PikPak 服务器上的文件，从技术上讲，PikPak 自己的工程师是有能力访问的。他们自己的隐私政策也直接写明了，会对上传内容进行审核，确保不包含"色情、侵权、仇恨、暴力或违法内容"。

换句话说，这不是一个"连服务商自己都看不到你数据"的那种隐私级产品。如果你有这个需求，Proton Drive（瑞士隐私法保护 + E2E 加密）或 Internxt Drive（去中心化架构）才是正确选择。

但对大多数普通用户来说，用 PikPak 存影视资源、工作文件，安全性完全够用，只是别把银行流水、身份证扫描件这类极度敏感的个人文件往上传就好。

## 隐私政策：他们会收集哪些数据？

PikPak 的隐私政策是公开可查的，收集范围包括：

- **账号信息**：注册邮箱、手机号、用户名
- **设备信息**：IP 地址、浏览器类型、设备标识符
- **使用行为**：你在云盘里的操作记录、下载历史、播放记录
- **位置信息**：用于判断你所在的地区（影响会员类型和定价）

这个收集范围，和主流云存储服务相比并无明显异常。Google Drive、Dropbox 收集的数据只多不少。

有一个实用建议：如果你对 IP 隐私有顾虑，可以通过 PikPak 的网页版访问——网页版通过 Cloudflare 反代，服务端只能看到 CF 的节点地址而非你的真实 IP。也可以搭配代理使用，App 端只会拿到代理 IP。
